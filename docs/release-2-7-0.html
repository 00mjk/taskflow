Release 2.7.0 (Master)

Taskflow 2.7.0 is the newest developing line to new features and improvements we continue to support. It is also where this documentation is generated. Many things are considered experimental and may change or break from time to time. While it may be difficult to be keep all things consistent when introducing new features, we continue to try our best to ensure backward compatibility.

Download

To download the newest version of Taskflow, please clone from Taskflow's GitHub.

Working Items

  • adding parallel sort tf::Taskflow::sort algorithms
  • enhancing the taskflow profiler (TFProf)

New Features

  • added tf::Executor::async to support asynchronously calling a function (see Run a Task Asynchronously)
    std::future<int> fu = executor.async([](){
    // do my work
    return 1;
    });
    // access the result through fu.get() or wait on the executor
    executor.wait_for_all();
    assert(fu.wait_for(std::chrono::seconds(1)) == std::future_status::ready);
  • added kernel algorithm, tf::cudaFlow::for_each,
    taskflow.emplace([](tf::cudaFlow& cf){
    // ... gpu tasks
    // assigns one to every element in my_gpu_data (type int*, 1000 elements)
    cf.for_each(my_gpu_data, 100, [] __device__ (int& item) {
    item = 1;
    });
    });
  • added kernel algorithm, tf::cudaFlow::for_each_index
    taskflow.emplace([](tf::cudaFlow& cf){
    // ... gpu tasks
    // assigns one to every element in my_gpu_data (type int*, 1000 elements)
    cf.for_each_index(0, 100, 1, [my_gpu_data] __device__ (int idx) {
    my_gpu_data[idx] = 1;
    });
    });
  • added kernel algorithm, tf::cudaFlow::transform
  • added explicit join method at tf::cudaFlow::join, tf::cudaFlow::join_n, tf::cudaFlow::join_until

Bug Fixes

There are no bug fixes in this release.

Deprecated Items

  • removed redundant methods, tf::Taskflow::broadcast, tf::Taskflow::precede, tf::Taskflow::succeed
  • removed tf::cudaFlow::predicate (replaced with tf::cudaFlow::join_until)
  • removed tf::cudaFlow::stream; the executor automatically determines a local, faster stream

  • added Governance
